# 예측 낙찰가 부정확 문제 분석 및 해결 방안

## 문제 요약
예측 낙찰가가 실제 낙찰가와 큰 차이가 남

**테스트 사례: 2025타경63180**
- 감정가: 172,000,000원
- 실제 최저입찰가: 84,280,000원 (3회 유찰 후 4회차)
- 실제 낙찰가: 92,600,000원
- 예측 시 표시된 경매회차: 2회차 ❌
- 예측 시 사용한 최저입찰가: 137,600,000원 (감정가*0.8) ❌

---

## 발견한 문제점

### 문제 1: 경매회차 계산 오류

**위치:** `main.py:254, 326`

```python
failure_count = int(badge_data.get('failure_count', 0))
...
"경매회차": failure_count + 1,
```

**실제 이력:**
- 1회차: 2025.11.11, 172,000,000원 (유찰)
- 2회차: 2025.12.16, 120,400,000원 (유찰)
- 3회차: 2026.01.27, 84,280,000원 (유찰)
- 4회차: 2026.02.03 예정 → 92,600,000원에 낙찰

ValueAuction API의 `failure_count` = 1로 반환 → `경매회차` = 2로 계산
실제 histories 개수 = 4

### 문제 2: 최저입찰가 고정값 사용 (핵심 문제)

**위치:** `main.py:977, 1077, 1203`

```python
def create_features(...):
    min_price = start_price * 0.8  # ❌ 항상 감정가의 80%로 고정

def create_features_v2(...):
    min_price = start_price * 0.8  # ❌

def create_features_enhanced(...):
    min_price = start_price * 0.8  # ❌
```

**영향:**
```
실제 최저입찰가:       84,280,000원  (감정가의 49%, 3회 유찰)
예측 시 사용한 값:    137,600,000원  (감정가의 80%, 항상 고정)
차이:                 53,320,000원  (63% 오차!)
```

**왜 문제인가?**
- 법원 경매는 유찰될 때마다 최저입찰가가 20%씩 감소
  - 1회: 100%
  - 2회: 80%
  - 3회: 64%
  - 4회: 51.2%
- 현재 코드는 **항상 80%로 고정**하여 유찰 횟수를 반영하지 않음
- 모델이 잘못된 최저입찰가로 예측 → 부정확한 결과

## 해결 방안

### 1. 경매회차 정확히 계산

**수정 전:**
```python
failure_count = int(badge_data.get('failure_count', 0))
경매회차 = failure_count + 1
```

**수정 후:**
```python
histories = matched_item.get('histories', [])
경매회차 = len(histories)  # histories 배열 길이로 직접 계산
```

### 2. 실제 최저입찰가 사용

#### 2-1. 최저입찰가를 API 응답에서 추출

**위치:** `main.py:280-290` (ValueAuction 데이터 파싱)

```python
# 최신 이력에서 최저입찰가 추출
lowest_bid_price = appraisal_price  # 기본값
if histories:
    latest_history = histories[-1]  # 가장 최근 이력
    price_data = latest_history.get('price', {})
    if isinstance(price_data, dict):
        lowest_bid_price = int(price_data.get('lowest_price', 0) or appraisal_price)
    else:
        lowest_bid_price = int(price_data or appraisal_price)
```

#### 2-2. 예측 함수에 최저입찰가 전달

**위치:** `main.py:1891-1898`

**수정 전:**

#### 2-3. create_features 함수 수정

**위치:** `main.py:966-1000, 1050-1150, 1176-1344`

**수정 전:**
```python
def create_features_enhanced(...):
    min_price = start_price * 0.8  # 항상 80% 고정
    ...
    features.extend([
        start_price,
        np.log1p(start_price),
        min_price,  # ❌ 고정값 사용
        np.log1p(min_price),
        0.8,  # ❌ 고정 비율
        ...
    ])
```

**수정 후:**
```python
def create_features_enhanced(..., lowest_bid_price: int = None):
    # 실제 최저입찰가 사용, 없으면 기본값 (감정가 * 0.8)
    min_price = lowest_bid_price if lowest_bid_price else start_price * 0.8
    # 최저가율 계산 (실제 비율)
    lowest_price_ratio = min_price / start_price if start_price > 0 else 0.8

    features.extend([
        start_price,
        np.log1p(start_price),
        min_price,  # ✅ 실제 최저입찰가
        np.log1p(min_price),
        lowest_price_ratio,  # ✅ 실제 비율 (0.49 등)
        ...
    ])
```
