## 2. AI 모델의 권리분석 변수 활용

### 확인 파일
`train_model_enhanced.py`

### 사용되는 권리 관련 특성 (총 10개)

#### 기본 특성 (4개) - Lines 157-162
1. `is_hard` - 권리분석복잡도 (0 or 1)
2. `tag_count` - 권리사항태그수 (숫자)
3. `np.log1p(tag_count)` - 태그수 로그 변환
4. `risk_score` - 리스크 점수 (is_hard * 0.2 + tag_count * 0.05)

#### 고급 특성 (6개) - Lines 259-279
5. `is_hard * np.log1p(start_price)` - 권리 복잡도 x 가격
6. `tag_density` - 태그 밀도 (tag_count / auction_round)
7. `total_risk` - 종합 리스크 점수
   ```python
   is_hard * 0.3 +
   tag_count * 0.1 +
   share_floor * 0.2 +
   share_land * 0.2 +
   debt_ratio * 0.2
   ```
8. `is_hard * debt_ratio` - 권리 x 부채 복합 리스크
9. `total_share` - 공유지분 종합 (share_floor + share_land)
10. `total_risk * auction_round` - 리스크 x 회차

### 모델 학습 시 사용되는 필드
```python
cursor.execute("""
    SELECT
        감정가, 물건종류, 지역, 면적, 경매회차, 입찰자수,
        입찰자수_실제, second_price, 권리분석복잡도, 권리사항태그수,
        공유지분_건물, 공유지분_토지, 청구금액비율,
        actual_price
    FROM predictions
""")
```

✅ **AI 모델은 권리분석 변수를 10가지 특성으로 변환하여 활용함**

## 3. 데이터 흐름

```
ValueAuction API
    ↓
badge.tags: ["대항력있는임차인", "선순위전세권"]

    ↓
원본데이터 필드에 badge 포함하여 저장

    ↓

- 🔴 빨간색: 대항력있는임차인, 선순위전세권, 가압류, 가처분, 가등기
- 🟡 노란색: 근저당권, 지상권, 유치권, 전세권
- 🔵 파란색: 기타 권리사항
```
