# Windows 작업 스케줄러 설정 가이드

주간 자동 데이터 수집 및 모델 재학습을 설정하는 방법입니다.

---

## 📋 설정 방법

### 1단계: 작업 스케줄러 열기

```
시작 메뉴 → "작업 스케줄러" 검색 → 실행
```

또는

```
Win + R → taskschd.msc 입력 → 확인
```

---

### 2단계: 기본 작업 만들기

1. 오른쪽 "작업 만들기" 클릭

2. **일반 탭**:
   - 이름: `경매 AI 주간 업데이트`
   - 설명: `매주 신규 데이터 수집 및 모델 재학습`
   - 보안 옵션:
     - ✅ 사용자의 로그온 여부에 관계없이 실행
     - ✅ 가장 높은 수준의 권한으로 실행

3. **트리거 탭**:
   - "새로 만들기" 클릭
   - 시작: `일정에 따라`
   - 설정:
     - ✅ 매주
     - 요일: `일요일` (선택)
     - 시간: `오전 3:00` (권장 - 서버 부하 적은 시간)
   - 고급 설정:
     - ✅ 사용

4. **동작 탭**:
   - "새로 만들기" 클릭
   - 동작: `프로그램 시작`
   - 프로그램/스크립트:
     ```
     C:\Users\unity\auction_gemini\scheduled_tasks\weekly_update.bat
     ```
   - 시작 위치(선택 사항):
     ```
     C:\Users\unity\auction_gemini
     ```

5. **조건 탭**:
   - 전원:
     - ☐ 컴퓨터의 AC 전원이 켜진 경우에만 작업 시작 (선택 해제 권장)
   - 네트워크:
     - ☐ 다음 네트워크 연결을 사용할 수 있는 경우에만 시작 (선택 해제)

6. **설정 탭**:
   - ✅ 요청 시 작업 실행 허용
   - ✅ 작업이 실패한 경우 다시 시작 간격: `1분`
   - 다시 시작 시도: `3`회
   - 실행 시간: `2시간`

7. **확인** 클릭하여 저장

## 🧪 테스트

### 즉시 실행 테스트:

1. 작업 스케줄러에서 방금 만든 작업 찾기
2. 오른쪽 클릭 → "실행"
3. 로그 파일 확인:
   ```
   C:\Users\unity\auction_gemini\logs\weekly_update_log.txt
   ```

### 로그 확인:

```powershell
# PowerShell에서 실행
Get-Content C:\Users\unity\auction_gemini\logs\weekly_update_log.txt -Tail 10
```

## 🔍 문제 해결

### 작업이 실행되지 않는 경우:

1. **권한 확인**:
   - 작업 스케줄러를 관리자 권한으로 실행했는지 확인

2. **경로 확인**:
   - 배치 파일 경로가 정확한지 확인
   - Python이 시스템 PATH에 등록되어 있는지 확인

3. **로그 확인**:
   ```
   작업 스케줄러 → 작업 기록 보기
   ```

4. **수동 테스트**:
   ```cmd
   cd C:\Users\unity\auction_gemini\scheduled_tasks
   weekly_update.bat
   ```
