# 이상치 자동 제거 가이드

데이터 품질 향상을 위해 파이프라인에 이상치 자동 제거 기능이 추가되었습니다.

---

## 🎯 기능 개요

### 문제
- 일부 비정상적인 데이터(예: 낙찰률 800% 초과)가 모델 정확도를 저하시킴
- 수동으로 이상치를 찾아 제거해야 했음

### 해결
- **자동**: 파이프라인 실행 시 이상치 자동 감지 및 제거
- **안전**: 극단적 이상치만 선별적 제거 (verified=0 처리)
- **추적**: 제거된 데이터 로그 기록

---

## 🔍 이상치 기준

### 자동 제거 대상

#### 1. 낙찰률 800% 초과
```
감정가: 11.1억원
실제 낙찰가: 91.2억원
낙찰률: 821.6% ❌ 제거 대상
```

**이유**: 감정가의 8배 이상 낙찰은 현실적으로 불가능 (데이터 오류 가능성)

#### 2. 오차율 100% 초과 (선택적)
```
예측: 5억원
실제: 15억원
오차율: 200% ❌ 제거 대상
```

**이유**: 모델이 완전히 예측 실패한 경우 (재학습에 악영향)

## 🛠️ 스크립트 설명

### `verify_outliers.py`
**기능**: 이상치 데이터 상세 분석

**주요 기능**:
- 오차율 30% 이상 데이터 조회
- 낙찰률 800% 초과 데이터 확인
- 중복 데이터 분석
- 처리 권장 사항 제시

**사용 시점**:
- 데이터 품질 점검 필요 시
- 이상치 제거 전 확인용

### `fix_outliers_safe.py`
**기능**: 이상치 데이터 안전 제거

**처리 방식**:
- 낙찰률 800% 초과 데이터만 `verified=0` 처리
- 데이터 삭제 없음 (추후 복구 가능)
- 제거 전후 통계 비교 출력

**안전 장치**:
- 사건번호/물건번호 체크 제거 (정상 데이터 오판 방지)
- 극단적 이상치만 선별적 제거
- 원본 데이터 보존

### `fix_outliers.py` (사용 중단)
**주의**: 너무 과격한 처리로 사용하지 않음
- 모든 사건번호 None 데이터 제거 → 전체 데이터 삭제 위험

## 📋 이상치 제거 전략

### 현재 전략: 보수적 접근
```python
# 낙찰률 800% 초과만 제거
if (actual_price / 감정가) * 100 > 800:
    verified = 0
```

**장점**:
- ✅ 명백한 오류만 제거 (안전)
- ✅ 정상 데이터 보존
- ✅ 복구 가능

### 향후 개선 방향

#### 1. 통계적 이상치 탐지
```python
# IQR 방식
Q1 = np.percentile(data, 25)
Q3 = np.percentile(data, 75)
IQR = Q3 - Q1
outlier_threshold = Q3 + 1.5 * IQR
```

#### 2. 머신러닝 기반 이상치 탐지
```python
from sklearn.ensemble import IsolationForest

detector = IsolationForest(contamination=0.01)
outliers = detector.fit_predict(data)
```

#### 3. 도메인 지식 기반 규칙
```python
# 예: 아파트 낙찰률 범위
if property_type == "아파트":
    if not (20 <= actual_rate <= 120):
        verified = 0
```
