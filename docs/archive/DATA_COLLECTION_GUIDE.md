# 데이터 수집 확대 가이드 (밸류옥션 API 사용)

고가 물건 데이터를 밸류옥션 API로 효율적으로 수집합니다.

---

## 🎯 목표

### 현재 상태
- **10억 초과 데이터**: 56건 (전체의 4.6%)
- **평균 오차율**: 1.34%
- **문제점**: 데이터 부족으로 고가 물건 예측 불안정

### 목표
- **10억 초과 데이터**: 200건 이상 (전체의 15%+)
- **추가 수집**: 약 150건
- **기대 효과**: 고가 물건 예측 안정성 대폭 향상

---

## ✨ 밸류옥션 API의 장점

기존 데이터 수집 방식을 그대로 사용하되, **고가 물건 필터**만 추가합니다.

### 장점
- ✅ **이미 낙찰된 물건** = 실제 낙찰가 포함
- ✅ **즉시 검증** = 예측 → 실제 비교 자동
- ✅ **높은 품질** = 검증된 데이터만 수집
- ✅ **완전 자동** = 수동 작업 불필요

### 기존 방식과의 차이
**기존**: 모든 가격대의 낙찰 물건 수집
**신규**: 감정가 10억 이상만 필터링하여 수집

---

## 🚀 빠른 시작

### 1단계: 고가 물건 수집 (완전 자동)

\`\`\`bash
python collect_high_price_valueauction.py --target 150
\`\`\`

**특징**:
- 밸류옥션 API에서 감정가 10억 이상 낙찰 물건만 가져옴
- 자동으로 예측 → 검증 수행
- 실제 낙찰가 포함 (수동 입력 불필요)

**예상 소요 시간**: 10~20분

---

### 2단계: 모델 재학습

\`\`\`bash
python auto_pipeline.py --skip-collect
\`\`\`

**자동 처리**:
1. ✅ 이상치 제거
2. ✅ 모델 재학습 (56건 → 200건+)
3. ✅ 전체 재예측
4. ✅ 구간별 분석
5. ✅ 서버 모델 리로드

## 🛠️ 문제 해결

### Q1: "API 응답 실패" 오류

**원인**: 밸류옥션 API 일시적 장애 또는 네트워크 문제

**해결**:
\`\`\`bash

python collect_high_price_valueauction.py --target 150
\`\`\`

---

### Q2: "더 이상 데이터가 없습니다" 메시지

**원인**: 밸류옥션에 10억 이상 낙찰 물건이 부족

**해결**:
\`\`\`bash
# 기준을 낮춰서 수집
python collect_high_price_valueauction.py --target 200 --min-price 500000000

# 또는 정기적으로 수집 (매주 새로운 낙찰 물건 추가됨)
\`\`\`

---

### Q3: 수집 속도가 느림

**원인**: API rate limiting

**해결**:
- 정상적인 동작입니다
- 밸류옥션 서버 부하 방지를 위한 딜레이 포함
- 예상 시간: 150건 수집에 10~20분
