# 웹 대시보드 개선 보고서

**작업 일시**: 2026-02-11
**문제 제기**: 고액 물건의 큰 오차가 평균을 왜곡하여 사용자 신뢰도 저하

---

## 🎯 문제 분석

### Before (문제점)
```
평균 오차금액: 20,729,818원 (약 2,070만원)
```

**사용자 반응**:
- 감정가 3억원 물건을 보는 사용자: "2천만원 오차? 너무 크네..." ❌
- 실제 3억원 물건 오차: ±56만원 (1~3억 구간 중위값)
- **96.1%가 평균에 의해 왜곡된 값**

### 원인
- 10억 초과 물건 31건의 평균 오차: **3억 8천만원**
- 이 소수의 고액 물건이 전체 평균을 끌어올림
- 평균값은 **이상치(outlier)에 매우 민감**

---

## 📊 감정가 구간별 실제 통계

### 중위값 기준 (대표성 높음)

| 구간 | 건수 | 오차율 | **중위 오차금액** | 평균 오차금액 |
|------|------|--------|------------------|--------------|
| 1억 이하 | 128건 | 2.1% | **±31만원** ✅ | ±73만원 |
| 1억~3억 | 412건 | 1.5% | **±56만원** ✅ | ±185만원 |
| 3억~5억 | 67건 | 3.9% | **±133만원** | ±660만원 |
| 5억~10억 | 72건 | 5.2% | **±367만원** | ±1,958만원 |
| 10억 초과 | 31건 | 7.5% | **±4,095만원** | ±3억 8천만원 |
| **전체** | **710건** | **2.5%** | **±80만원** ✅ | **±2,073만원** ❌ |

### 핵심 발견
```
중위 오차금액:    804,281원 (약 80만원) ✅ 실제 대표값
평균 오차금액: 20,729,818원 (약 2,070만원) ❌ 왜곡된 값

중위값이 평균보다 96.1% 낮음!
```

## ✅ 개선 사항

### 1. Database 개선
**파일**: `database.py`

**변경 내용**:
- `get_accuracy_stats()` 함수에 `median_error_amount` 추가
- NumPy를 사용한 중위값 계산

```python
# 중위값 계산
cursor.execute("""
    SELECT error_amount
    FROM predictions
    WHERE verified = 1 AND error_amount IS NOT NULL
    ORDER BY error_amount
""")
error_amounts = [r[0] for r in cursor.fetchall()]
median_error_amount = int(np.median(error_amounts))

stats['median_error_amount'] = median_error_amount  # 추가
```

## 🎯 효과

### 1. 사용자 신뢰도 향상
**Before**:
- 감정가 3억원 물건 사용자가 "평균 오차 2천만원" 보고 불안
- 실제로는 ±56만원인데 말이죠

**After**:
- "1억~3억 구간: ±56만원" 직접 확인
- "일반적으로 80만원 정도 오차" → 신뢰 가능

### 2. 투명성 향상
- 감정가 구간별로 차이가 있음을 명확히 표시
