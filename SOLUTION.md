# AI 예측 모델 문제 해결 방안

## 문제 상황
AI 모델이 다음 변수들을 예측에 반영하지 않음:
- ❌ 지역 (서울/인천/경기/부산/대구 모두 동일한 결과)
- ❌ 물건종류 (아파트/오피스텔/다세대/단독주택 모두 동일)
- ❌ 면적 (59㎡/84㎡/110㎡/150㎡ 모두 동일)

정상 작동하는 변수:
- ✅ 경매회차 (1회차 → 2회차 → 3회차로 갈수록 낮아짐)
- ✅ 감정가 (금액에 비례하여 변화)

## 원인 분석

현재 모델이 이런 결과를 보이는 이유는 다음 중 하나:

1. **학습 데이터 부족**: 실제 낙찰 데이터가 없어서 더미 데이터로 학습했을 가능성
2. **Feature 가중치 문제**: 모델이 지역/물건종류/면적의 가중치를 거의 0으로 학습
3. **데이터 편향**: 학습 데이터에서 대부분이 서울 아파트여서 다른 지역/물건의 패턴을 학습하지 못함

## 해결 방법

### ✅ 방법 1: 통계 기반 조정 계수 추가 (즉시 적용 가능)

AI 모델 예측값에 지역별/물건별 통계 기반 조정을 추가합니다.

**장점:**
- 즉시 적용 가능
- 모델 재학습 불필요
- 실제 통계 데이터 기반

**단점:**
- 고정된 조정 계수 사용 (유연성 부족)

**구현 방법:**
```python
# main.py에 추가

# 지역별 조정 계수 (서울 기준 1.0)
REGION_ADJUSTMENT = {
    "서울": 1.05,      # +5%
    "경기": 0.98,      # -2%
    "인천": 0.95,      # -5%
    "부산": 0.92,      # -8%
    "대구": 0.88,      # -12%
    "대전": 0.90,
    "광주": 0.87,
    "울산": 0.93,
    "세종": 0.94,
    "기타": 0.85       # -15%
}

# 물건종류별 조정 계수 (아파트 기준 1.0)
PROPERTY_ADJUSTMENT = {
    "아파트": 1.0,      # 기준
    "오피스텔": 0.92,   # -8%
    "다세대": 0.85,     # -15%
    "단독주택": 0.78,   # -22%
    "상가": 0.82,       # -18%
    "토지": 0.75        # -25%
}

# 면적별 조정 (㎡당 프리미엄)
def get_area_adjustment(area: float) -> float:
    """면적에 따른 조정 계수"""
    if area < 60:  # 소형 (18평 미만)
        return 0.95
    elif area < 85:  # 중소형 (25평 미만)
        return 1.0
    elif area < 110:  # 중형 (33평 미만)
        return 1.02
    else:  # 대형 (33평 이상)
        return 1.05

# predict_price_advanced 함수 수정
def predict_price_advanced(...):
    # ... 기존 모델 예측 ...
    predicted = model.predict(features)[0]

    # 통계 기반 조정 적용
    region_factor = REGION_ADJUSTMENT.get(region, 0.9)
    property_factor = PROPERTY_ADJUSTMENT.get(property_type, 0.9)
    area_factor = get_area_adjustment(area)

    # 조정된 예측값
    adjusted_predicted = predicted * region_factor * property_factor * area_factor

    return int(adjusted_predicted)
```

### ✅ 방법 2: 실제 낙찰 데이터로 모델 재학습 (근본적 해결)

**필요한 것:**
1. 실제 경매 낙찰 데이터 (최소 1,000건 이상)
2. 다양한 지역/물건종류 포함
3. 낙찰가, 감정가, 지역, 물건종류, 면적, 경매회차 등 포함

**구현 순서:**
1. 데이터 수집: ValueAuction API나 공공데이터에서 수집
2. 데이터 정제: 이상치 제거, 결측치 처리
3. Feature Engineering: 지역/물건종류 원-핫 인코딩
4. 모델 학습: XGBoost, LightGBM 등
5. 검증: 지역별/물건별 예측 차이 확인

### ✅ 방법 3: 하이브리드 접근 (권장)

AI 모델 + 통계 조정을 결합합니다.

**전략:**
1. AI 모델로 기본 예측 (감정가, 경매회차 중심)
2. 통계 기반으로 지역/물건종류/면적 조정
3. 실제 낙찰 데이터 누적되면 점진적으로 모델 개선

## 추천 솔루션

**장기 (1-2개월)**: 충분한 데이터로 모델 재학습
- 지역별/물건별 패턴이 반영된 정확한 모델
- 지속적인 업데이트 체계 구축

## 기대 효과

통계 기반 조정 적용 후:
- 지역별 예측 차이: 서울 3.15억 → 인천 3.0억 (5% 차이)
- 물건종류별 차이: 아파트 3.15억 → 다세대 2.7억 (15% 차이)
- 면적별 차이: 59㎡ 3.0억 → 110㎡ 3.2억 (7% 차이)

사용자 경험 개선:
- 지역/물건 선택이 의미있는 결과 제공
- 예측 신뢰도 향상
